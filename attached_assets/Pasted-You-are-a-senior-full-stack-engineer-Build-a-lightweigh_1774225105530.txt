👤 USER SYSTEM (SIMPLIFIED)
====================================

Use a very lightweight admin + ownership model.

Admin:
- Define one admin email in a config file
- Use a lightweight pseudo-login with a simple email input or local session
- If current email matches admin email, treat user as admin

Do NOT implement:
- full authentication provider
- signup flow
- user accounts table
- email storage in database

Ownership model for normal users:
- Do not store user emails in the database
- Generate a browser-local owner token and store it in localStorage
- Example key: aarrr_owner_token
- If token does not exist, create a random unique token
- When a dashboard is created, save createdByToken using that local token
- A dashboard can be edited only if:
  - current browser token == dashboard.createdByToken
  - OR current user is admin

Permissions:

All users:
- view projects
- view dashboards
- create dashboards inside a project

Dashboard edit:
- allowed only for the creator in the same browser (matching createdByToken)
- OR admin

Admin only:
- create project
- edit project
- delete project
- delete dashboard

Important:
- do not store user emails in projects or dashboards
- do not create a users table
- ownership must rely on localStorage token only
- admin remains email-based, but normal user ownership should not depend on email
